# Rules Distillation Playbook

> If the request is general, ambiguous, or touches many files — ask clarifying yes/no questions **before acting**. Use numbered bullets with explicit answer format: e.g. `1-yes/no  2-A/B`.

Use this when promoting repeated principles from skills and references into stronger shared rules.

## Workflow

1. Read the relevant skills, references, and current rule files together.
2. Extract repeated principles that clearly affect future assistant behavior.
3. Compare the candidates against existing rules to avoid duplicate guidance in new wording.
4. Decide whether each principle should be appended, revised, added as a new section, or left at the skill level.
5. Draft concise rule text and present it for review instead of treating the rule change as automatic.

## Output expectation

- sources reviewed
- repeated principles found
- proposed rule action
- target file or section
- draft text
- over-generalization risks
