# Product Discovery Playbook

> If the request is general, ambiguous, or touches many files — ask clarifying yes/no questions **before acting**. Use numbered bullets with explicit answer format: e.g. `1-yes/no  2-A/B`.

Use this when shaping a feature, product, or startup idea.

## Workflow

1. Clarify the user, pain, and trigger.
2. Define the smallest useful workflow.
3. Identify assumptions that must be true.
4. Separate must-have scope from expansion ideas.
5. Convert vague intent into testable acceptance criteria.
6. Identify delivery, adoption, and operational risks.

## Output expectation

- crisp problem statement
- target user
- workflow
- acceptance criteria
- risks and assumptions
- recommended next slice
