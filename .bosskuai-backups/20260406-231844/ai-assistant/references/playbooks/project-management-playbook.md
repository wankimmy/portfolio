# Project Management Playbook

> If the request is general, ambiguous, or touches many files — ask clarifying yes/no questions **before acting**. Use numbered bullets with explicit answer format: e.g. `1-yes/no  2-A/B`.

Use this when the task is about execution control, team coordination, ownership, sequencing, risk tracking, or keeping a project on track.

## Workflow

1. Clarify the target outcome, owner, timeframe, and definition of done.
2. Break work into milestones, slices, and handoff points.
3. Identify dependencies, blockers, and decisions that could stall execution.
4. Assign review cadence, operating rhythm, and decision checkpoints.
5. Recommend the smallest next action that improves delivery confidence.

## Output expectation

- project outcome
- owners and responsibilities
- milestone plan
- risks and blockers
- decision cadence
- next operational step
