# i18n / l10n Playbook

> If the request is general, ambiguous, or touches many files — ask clarifying yes/no questions **before acting**. Use numbered bullets with explicit answer format: e.g. `1-yes/no  2-A/B`.

Use this when reviewing localization readiness, locale handling, or multilingual UX.

## Workflow

1. Read the translation surface, locale config, and formatting helpers.
2. Map locale selection, fallback, and intentionally untranslated areas.
3. Review formatting, pluralization, expansion, and RTL behavior.
4. Review translation ownership, QA, and release workflow.
5. Recommend the smallest localization slice that reduces future product pain.

## Output expectation

- localization summary
- fallback and formatting behavior
- layout and translation risks
- intentional gaps
- recommended next localization slice
