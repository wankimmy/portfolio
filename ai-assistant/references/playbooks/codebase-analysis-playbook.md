# Codebase Analysis Playbook

> If the request is general, ambiguous, or touches many files — ask clarifying yes/no questions **before acting**. Use numbered bullets with explicit answer format: e.g. `1-yes/no  2-A/B`.

Use this when reading an unfamiliar repository or explaining how the source code actually works.

## Workflow

1. Identify the stack, entry points, and folder structure.
2. Locate the real execution path for the feature or flow in question.
3. Distinguish generated files, adapters, frameworks, and business logic.
4. Verify assumptions against source code and tests.
5. Summarize the architecture and behavior in plain language.

## Output expectation

- repo and stack summary
- execution path
- important modules
- conventions
- open uncertainties
