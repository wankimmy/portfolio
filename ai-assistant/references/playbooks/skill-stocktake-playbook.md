# Skill Stocktake Playbook

> If the request is general, ambiguous, or touches many files — ask clarifying yes/no questions **before acting**. Use numbered bullets with explicit answer format: e.g. `1-yes/no  2-A/B`.

Use this when auditing the quality and maintainability of local skills and related commands.

## Workflow

1. Inventory the relevant skills, commands, and linked references in the current workspace.
2. Review them for trigger clarity, uniqueness, overlap, freshness, and maintenance cost.
3. Compare against `AGENTS.md`, shared memory, and references so duplicated guidance is not mistaken for unique value.
4. Assign a practical verdict: keep, improve, merge, update, or retire.
5. Present the smallest safe maintenance actions first rather than attempting a full rewrite.

## Output expectation

- scope audited
- items reviewed
- verdict per item
- justification
- recommended next maintenance slice
