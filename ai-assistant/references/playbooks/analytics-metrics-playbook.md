# Analytics / Metrics Playbook

> If the request is general, ambiguous, or touches many files — ask clarifying yes/no questions **before acting**. Use numbered bullets with explicit answer format: e.g. `1-yes/no  2-A/B`.

Use this when defining instrumentation, KPI logic, funnels, or experiment measurement.

## Workflow

1. Clarify the decision or hypothesis the measurement must support.
2. Read the current events, dashboards, and analytics code or docs.
3. Map event grain, identities, funnel stages, and attribution paths.
4. Review data quality, privacy, and validation risks.
5. Recommend the smallest instrumentation slice that supports a real decision.

## Output expectation

- measurement goal
- event and funnel model
- attribution/data quality risks
- privacy considerations
- instrumentation and validation plan
