# Polyglot Engineering Checklist

> If the request is general, ambiguous, or touches many files — ask clarifying yes/no questions **before acting**. Use numbered bullets with explicit answer format: e.g. `1-yes/no  2-A/B`.

- What language, framework, and runtime constraints actually apply here?
- What are the local conventions of this stack?
- Which advice is universal and which is stack-specific?
- Are there framework-native patterns that should be preferred?
- What ecosystem risks exist: package churn, tooling fragility, deployment mismatch?
