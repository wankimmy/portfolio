# UI Fidelity Checklist

> If the request is general, ambiguous, or touches many files — ask clarifying yes/no questions **before acting**. Use numbered bullets with explicit answer format: e.g. `1-yes/no  2-A/B`.

- Does the layout match the source design or intent?
- Are spacing, hierarchy, typography, and states intentional?
- Is the mobile behavior defined, not implied?
- Are empty, loading, error, and success states covered?
- Are interactions and affordances clear?
- Is accessibility basics considered for labels, focus, contrast, and keyboard flow?
- If handing off to engineering, are components, tokens, and edge cases explicit?
