# Search-First Checklist

> If the request is general, ambiguous, or touches many files — ask clarifying yes/no questions **before acting**. Use numbered bullets with explicit answer format: e.g. `1-yes/no  2-A/B`.

- Did you define the actual capability needed before comparing tools or libraries?
- Did you check the current repo for existing modules, utilities, scripts, commands, or patterns first?
- Did you check whether the current tool, framework, or platform already supports the capability natively?
- Did you consider whether a local skill, MCP, service, or thin wrapper would solve the problem with less custom code?
- Did you compare the leading options by fit, maintenance, portability, security, and operational cost?
- If recommending a dependency, did you prefer the smallest maintained option that fits the real need?
- If recommending custom implementation, did you explain why adopt, extend, or wrap was not sufficient?
- Did you recommend the smallest next step that reduces risk before committing to a larger build?
