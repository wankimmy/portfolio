# Customer Discovery Checklist

## Pre-work
- [ ] Discovery goal defined: validate problem / explore workflow / test solution concept
- [ ] Target segment specified (not "everyone")
- [ ] 5–8 participants recruited (mix of ideal customers + skeptics)
- [ ] Interview guide written (open-ended questions only — no leading)
- [ ] Recording consent obtained

## During interview
- [ ] Opened with context: "Tell me about the last time you..."
- [ ] Let participant lead — no solution pitching
- [ ] Followed up on emotions: "How did that make you feel?"
- [ ] Asked about workarounds and current tools
- [ ] Explored frequency and severity of the problem
- [ ] Closed with referral ask: "Who else should I talk to?"

## After interview
- [ ] Notes captured within 1 hour while fresh
- [ ] Tagged by theme: [pain] [workaround] [frequency] [budget] [stakeholder]
- [ ] Direct quotes captured verbatim (with permission)
- [ ] JTBD framing noted: "When I ___, I want to ___, so I can ___"

## Synthesis
- [ ] Patterns across ≥3 interviews documented
- [ ] Top 3 pains ranked by frequency × severity
- [ ] Personas drafted from cluster of similar profiles
- [ ] Assumptions updated: confirmed / invalidated / new
- [ ] Product implications listed with supporting evidence

## Quality gate
- [ ] No confirmation bias: disconfirming evidence included
- [ ] Sample size sufficient for decision confidence (minimum 5 interviews per segment)
- [ ] Findings shared with team before decisions made
